      • JDBC

        • Java Data Base Connectivity : It's defined as

        Five Steps of JDBC Connectivity -


        1. Step 1 : Load Driver / Server

          2. For loading the driver file first import the dependency Driver file from maven project,this dependency add in dependencies tag .

          We are loading the Driver file in PostgreySql by using the method name forName()

          • forName() method is Static method
          • It's present in classname Class
          • It's static method that why it's called by it's class name Class.forName()
          • Class.forName("org.postgresql.Driver");

        2. Step 2 : Create / Initialize Connection

          3. To Create connection we have to provide URl , Username , password

          • Connection :
          • It's an a interface present in java.sql package.
          • To create the implementation object for connection interface we have to use getConnection() method.
          • getConnection ("Url" , "UserName" , "Password");
          • It is a static method present in DriverManager class (Helper class).
          • It can return connection object after taking url , UserName , Password .
          • If given information is false then throws SQLException
          • getConnection method have two approaches-
          • i> getConnection("Url","UserName","password"); ---- getConnection("jdbc:postgresql://localhost:5432/qsp","postgres","root");
          • ii> getConnection("url"); ---- Connection con = DriverManager.getConnection("jdbc:postgresql://localhost:5432/qsp?user=postgres&password=root");

        3. Create Statement

          4. We can create Statement by using Three interfaces .

          1. Statement Interface (Car)

            2. But we have to uses the Statement Interface

            • Statement interface is used for to store static data i.e manual data insertion .
            • Statement interface present in java.sql package
            • We uses the non static method of Interface Connection
            • It can throws exception like SQLException
            • Statement ste = con.createStatement() ;

          2. PreparedStatement Interface (Cycle)

            This interface is used for give the Dynamically from user.

            • PreparedStatement interface present in java.sql Package.
            • We can uses the non static method of object of Connection interface.
            • Method name is prepareStatement("SQL queries.")
            • For fetching data we want to create the ResultSet interface object.

          3. CallableStatement Interface (Horse)

        4. Execute SQL Queries

        5. Close Connection

          6. We can close connection by using close() method which is present in Connection interface

          • What is necessary to close connection ?
          • Ans : To increase the performance .
          • When we are creating connection , it will uses system resources that will make application slow, to increase application performance we use close connection.
          1. execute()
          2. executeUpdate()
            • It is an non-static method present in Statement interface .
            • Syntax public int executeUpdate("DML");
            • It is used to execute only DML queries.
            • It will return no of rows affected by executing DML queries(insert , update , delete)
            • Eg: int n = stm.executeUpdate("insert into student values(45,'Rohit'),(33,'Hardik')");
          3. executeQuery()
            • It is an non-static method present in Statement interface.
            • Syntax: public ResultSet executeQuery("DQL");
            • We can execute only DQL queries (Select queries).
            • It will return ResultSet .
            • ResultSet:
              • Resultset :
              • It is an interface present in java.sql package.
              • It is used to fetch data from database.
              • It is acts like cursor & it points to a table.
              • It contains two methods
              • 1> next();
              • 2> getx();
              • We can create implementation object for resultset by using
                1. getResultSet();
                2. executeQuery();
