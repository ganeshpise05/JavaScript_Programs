  4. Abstraction

    5. It is process of hiding the implementation detail & only showing features is known as Abstraction. It can be achieved by using the help of Classes & Interfaces

    Abstract Methods : Abstract methods are methods that are declared without an implementation. They do not have a body and are meant to be overridden by subclasses.

    Concrete (Non-Abstract) Methods : Concrete methods are methods that have a complete implementation (i.e., they have a body with code). These methods can be used directly by subclasses or overridden.

    We can not Create Object of abstract class.

    When class having one abstract method then it is mandatory to create class as Abstarct.


  5. Interface

    6. It is extended version of Abstraction

    The interface in which only abstract methods are present concrete non-static methods are not allowed here.

    Functonal Interface: The interface which contain only one Abstarct method is known as Functonal Interface.

    Marker Interface: The interface which does not contain abstarct Method is known as Marker interface.

    Extends: When Class inherit properties of Abstarct class then we uses extends keyword.

    When one interface inherit properties of another interface then we uses extends keyword.


    Implements: When Class inherit properties of Interface class then we uses impliments keyword.
