Promises Theory

  • Promises object in Js.
  • Used to handle asynchronous task.
  • with promises there are 3 states : 1> pending , 2> resolve , 3> rejected
    1. pending:
      • promises will be in pending state when it is neither resolved or rejected .

    2. resolve:
      • When promises are fulfilled it will be in resolve state.
      • When promise is resolve it will call then block.
      • And we can be used to handle the errors or exception

    3. rejected:
      • When promises are not fullfilled it will be in rejected state.
      • When promises is rejected it will be call the catch block,
      • catch block can be used to handle the error or exception.

Note : Your promises either resolved or rejected at a same time .

Asynchronous task in Js will internally return the promise we just have to handle this promise using then & catch block

fetch() return promise it handle by using then & catch block.
